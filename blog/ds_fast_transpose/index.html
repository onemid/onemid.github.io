Sparse Matrix Fast Transpose

August 29, 2018 Gary Gong

18 minute read

Overview

Sparse Matrix 又名稀疏矩陣，意思是一個矩陣超多零；使用一般的矩陣註記方法，有浪費空間之虞，所以產生了列表式的註記法，如以下舉例：

我們給定一個稀疏矩陣$ \boldsymbol{M}$，其中矩陣內容為： $$ \boldsymbol{M} = \begin{bmatrix} 15 & 0 & 0 & 22 & 0 & -15 \newline 0 & 11 & 3 & 0 & 0 & 0 \newline 0 & 0 & 0 & -6 & 0 & 0 \newline 0 & 0 & 0 & 0 & 0 & 0 \newline 91 & 0 & 0 & 0 & 0 & 0 \newline 0 & 0 & 28 & 0 & 0 & 0 \newline \end{bmatrix} $$ 轉化為稀疏矩陣記法為：

# Row Col Val
a[0] 6 6 8
a[1] 0 0 15
a[2] 0 3 22
a[3] 0 5 -15
a[4] 1 1 11
a[5] 1 2 3
a[6] 2 3 -6
a[7] 4 0 91
a[8] 5 2 28

其中 a[0] 記錄著 $\boldsymbol{M}$ 矩陣有幾個 row 幾個 column 和非零 value 的總數；所以，解釋下來為：此 $\boldsymbol{M}$ 為 6 乘 6 的矩陣，其中非零值有 8 個；而事實上這種註記方法可以節省掉不少的空間。

在此我們用C的struct去註記這個稀疏矩陣記法：

typedef struct {
	int row;
	int col;
	int val;
} term;

這樣的話，a[6].row為 2、 a[6].col為 3 ，以此類推。

Typical Sparse Matrix Transpose

最基礎的矩陣轉置其實就是 row 與 column 之間的 value 交換，也是我們在手寫矩陣轉置運算時，最為直覺的方法，我們重用矩陣 $\boldsymbol{M}$ ，我們註記轉置後的矩陣為 $\boldsymbol{M}^T$：

$$ \boldsymbol{M}^T = \begin{bmatrix} 15 & 0 & 0 & 0 & 91 & 0 \newline 0 & 11 & 0 & 0 & 0 & 0 \newline 0 & 3 & 0 & 0 & 0 & 28 \newline 22 & 0 & -6 & 0 & 0 & 0 \newline 0 & 0 & 0 & 0 & 0 & 0 \newline 15 & 0 & 0 & 0 & 0 & 0 \newline \end{bmatrix} $$ 我們用最基礎的數學註記法，其中： $$ \ [{\boldsymbol{M}^T}{ij}] = [\boldsymbol{M}]{ji} $$ 亦即，比如說我們有一個「4」這個值，放在矩陣 $i=5$，$j=8$的位置，那他的轉置後的位置就在$i=8$，$j=5$。但是在稀疏矩陣記法內，我們要保證每一筆資料在 row index 保持遞增的情形，在數筆資料有相同 row index 時，要保持著 column index 遞增情形，而非單純的 i 、 j 值交換；據此，我們有一套流程：

void transpose(term a[], term b[]){
    b[0].row = a[0].col;
    b[0].col = a[0].row;
    b[0].val = a[0].val;
    int current_record = 1;
    for (int i = 0; i < a[0].col; i++) {
        for (int j = 1; j <= a[0].val; j++) {
            if (a[j].col == i)	{
                b[current_record].row = a[j].col;
                b[current_record].col = a[j].row;
                b[current_record].val = a[j].val;
                current_record++;
            }
        }
    }
}

這個程式大概是這樣跑的：

a[0] | 6              | 6                 | 8                |
a[1] | 0              | 0                 | 15               |
a[2] | 0              | 3                 | 22               |
a[3] | 0              | 5                 | -15              |
a[4] | 1              | 1                 | 11               |
a[5] | 1              | 2                 | 3                |
a[6] | 2              | 3                 | -6               |
a[7] | 4              | 0                 | 91               |
a[8] | 5              | 2                 | 28               |

那個i

/* 第一層迴圈，我們記作 first_loop */
for  (int i = 0; i < a[0].col; i++) {
...
}

其實就是我現在要讀取哪一個 column 當成我的 row；比如說 i 就是我現在要瞄準 $\boldsymbol{M}$ 第 i 個 row 要寫進 $\boldsymbol{M}^T$ 的第 i 個 column。

那個j

/* 第二層迴圈，我們記作 second_loop */
for  (int j = 1; j <= a[0].val; i++) {
...
}

就是說我每一個值（也就是a[1]到a[8]）都要看過，為啥要看過，要看下面的if在幹嘛：

/* 第二層迴圈裡面，我們記作 second_loop_inside */
if (a[j].col == i)	{
    b[current_record].row = a[j].col;
    b[current_record].col = a[j].row;
    b[current_record].val = a[j].val;
    current_record++;
}

就是說，因為我要保證我轉置出來的 b的資料 row 都是以遞增形式增長，所以我要特別挑過我要寫的東西，我a[j].col剛剛好是我鎖定的第 i 個 row，然後逐一檢查每筆資料的 col 是不是剛好我目前所想要的 col 值。

比如說我現在要用a第 0 (i為0時) 個 column 進行轉置成b的 row，那我就只看 a[j].col == 0 的部分（也就是這裡if的用處），我也不先看 col 為1還是4還是怎樣，我就只看0。

那我就進行交換寫入到b[current_record]裡面，這裡保證我轉置出來的 b的資料 row 都是以遞增形式增長。

但是這樣有兩層迴圈，在分析上時間複雜度最差可以到平方項，所以有更快的方法，稱之為 Fast Matrix Transpose。

Fast Sparse Matrix Transpose

這個演算法利用以空間換取時間的方式，壓縮本來 Typical Sparse Matrix Transpose 方法所需時平方項次的時間複雜度。簡單來說，在這個演算法裡面，我們利用額外的註記，去加速我們的運算。

演算法 b[] 行為模式 a[] 行為模式
Typical 按照順序寫 鎖定第 i 個 col，逐條挑出 a[].col == i 者，轉置，寫入 b[] 內。
Fast 跳著寫 逐條看，看了之後轉置，去看看要寫去 b[] 的哪邊

在 Fast 演算法時，我看完 a[] 其實就已經寫完 b[] 了；而 Typical 我還不知道要看幾遍 a[] 才能寫完 b[]。

要如何實現「跳著寫」這個問題，我們就必須額外開一張表去註記「我們這個東西究竟要寫去哪裡」。

所以說，在此，我們開兩個矩陣去註記一些東西：nonZeroRow和startingPos

nonZeroRow是註記每一個 col 裡面，有幾個 row 是有非零值的。

startingPos是註記這個 col 究竟是要從哪一個 b index 開始寫起。

據此幾個概念，我們形成了一些程式碼：

void fastTranspose(term a[], term b[]) {
    int nonZeroRow[MAX_COL] = {0}, startingPos[MAX_COL] = {0};
    b[0].col = a[0].row;
    b[0].row = a[0].col;
    b[0].val = a[0].val;
    for (int i = 1; i <= a[0].val; i++) {
        nonZeroRow[a[i].col]++;
    }
    startingPos[0] = 1;
    for (int i = 1; i <= a[0].col; i++) {
        startingPos[i] = startingPos[i-1] + rowTerms[i-1];
    }
    for (int i = 1; i <= a[0].val; i++) {
        j = startingPos[a[i].col]++;
        b[j].row = a[i].col;
        b[j].col = a[i].row;
        b[j].val = a[i].val;
    }
}

首先，第一隻迴圈

for (int i = 1; i <= a[0].val; i++) {
    nonZeroRow[a[i].col]++;
}

我們是要計算，在這個 col 底下有幾個 row 是非零值。

而 staringPos[0] = 1代表，我從b[1]開始寫起（b[0]是註記稀疏矩陣資訊，不能寫的）

startingPos[0] = 1;

第二隻迴圈，是要註記每一個 col 個別要寫到哪一個 b 的位置

for (int i = 1; i <= a[0].col; i++) {
    startingPos[i] = startingPos[i-1] + nonZeroRow[i-1];
}

每一個 startingPos 都是由上一個 startingPos位置 和 上一個nonZeroRow的個數加總起來的新位置；也就是，假若上一個startingPos位置為3那我上一個 col 的nonZeroRow有4個，此時，上一次的寫入就是已經被占用3、4、5、6的位置，基本上你的新位置就是起始點3再跳4格，所以是7。

接下來，第三隻迴圈：

for (int i = 1; i <= a[0].val; i++) {
    j = startingPos[a[i].col];
    b[j].row = a[i].col;
    b[j].col = a[i].row;
    b[j].val = a[i].val;
    startingPos[a[i].col]++;
}

[A Note about i++ and ++i]

首先取出，目前這個 col 應該寫在 b 的哪裡，然後進行轉置寫入。

接下來根據這個 col 的 startingPos 再往前加 1 上去（也就是我們日後要寫到相同的 col 時，新的位置應該在哪）。
